Install

pip install claude-swarm
brew tap laurenalexander2/tap && brew install claude-swarm

Requires Python 3.11+.

Human Commands

These are the commands you run. Everything else is handled by the AI sessions.

ao start

Usage: ao start [OPTIONS] DESCRIPTION

  Launch orchestrator mode in Claude Code.

Options:
  --project-dir TEXT  Project directory
$ ao start "Build a REST API with auth and rate limiting"

This explores your codebase, proposes a plan for splitting work across sessions, waits for your approval, then prints a prompt for each session. You paste each prompt into a separate Claude Code terminal. The orchestrator monitors everything from there.

Example

Launch the orchestrator:

$ ao start "Build a REST API with auth and rate limiting"

The orchestrator explores your codebase, then presents a plan:

Here's my proposed decomposition:

  Session A — Backend API
  Task: Build Express routes, auth middleware, rate limiter
  Files: src/api/, src/middleware/
  Dependencies: none

  Session B — Frontend
  Task: Build React UI with login and dashboard
  Files: src/components/, src/pages/
  Dependencies: needs A's auth types

Approve this plan, or tell me what to change.

You approve (or ask for changes). The orchestrator initializes sessions and prints a prompt for each one. You paste each prompt into a separate Claude Code terminal.

From there, sessions coordinate automatically via the SQLite bus:

A
$ claude-swarm context add "All API responses use {data, error} envelope" --session A --category decision
B
$ claude-swarm sync --session B
[SYNC] 1 new context
CTX [decision] by=A: "All API responses use {data, error} envelope"
B
$ claude-swarm message A "What's the auth token format?" --from B
A
$ claude-swarm sync --session A
[SYNC] 1 new message(s)
MSG from=B: "What's the auth token format?"

$ claude-swarm reply 1 "Bearer JWT in Authorization header" --from A
A
$ claude-swarm commit "add user auth endpoints" --session A
$ claude-swarm review request --from A --to orchestrator --diff "$(git diff main)"
A
$ claude-swarm merge-ok A
A is clear to merge
$ claude-swarm push --session A

Agent Commands

These are used by the AI sessions to coordinate. You don't need to run these yourself.

Usage: claude-swarm [OPTIONS] COMMAND [ARGS]...

Options:
  --version  Show the version and exit.
  --db TEXT  Path to bus database
  --help     Show this message and exit.

sync

Usage: claude-swarm sync [OPTIONS]

  Sync: check inbox, read new context, heartbeat. Silent when nothing new.

Options:
  --session TEXT  Session ID to sync  [required]

context add

Usage: claude-swarm context add [OPTIONS] BODY

  Add a shared context entry.

Options:
  --session TEXT                  Session ID  [required]
  --category [decision|interface|warning|convention|discovery]
                                  Context category  [required]

context show

Usage: claude-swarm context show [OPTIONS]

  Show all shared context entries.

message

Usage: claude-swarm message [OPTIONS] TO_ID BODY

  Send a message to another session.

Options:
  --from TEXT  Sender session ID  [required]

inbox

Usage: claude-swarm inbox [OPTIONS]

  Check unread messages for a session.

Options:
  --session TEXT  Session ID  [required]

reply

Usage: claude-swarm reply [OPTIONS] MESSAGE_ID BODY

  Reply to a message (marks original as read and sends reply).

Options:
  --from TEXT  Sender session ID  [required]

init

Usage: claude-swarm init [OPTIONS]

  Initialize the orchestrator database and register sessions.

Options:
  --sessions TEXT  Space-separated "ID:workstream" pairs  [required]

status

Usage: claude-swarm status [OPTIONS]

  Show all sessions and their status.

update

Usage: claude-swarm update [OPTIONS] SESSION_ID

  Update a session's status or note.

Options:
  --status TEXT  New status
  --note TEXT    Status note

claim

Usage: claude-swarm claim [OPTIONS] FILE_PATH

  Claim a file for exclusive editing.

Options:
  --session TEXT  Session ID  [required]

claims

Usage: claude-swarm claims [OPTIONS]

  List all file claims.

unclaim

Usage: claude-swarm unclaim [OPTIONS] FILE_PATH

  Release a file claim.

Options:
  --session TEXT  Session ID  [required]

review request

Usage: claude-swarm review request [OPTIONS]

  Request a review from another session.

Options:
  --from TEXT  Requester session ID  [required]
  --to TEXT    Reviewer session ID  [required]
  --diff TEXT  Diff content  [required]

review list

Usage: claude-swarm review list [OPTIONS]

  List all reviews.

review show

Usage: claude-swarm review show [OPTIONS] REVIEW_ID

  Show details of a specific review.

review approve

Usage: claude-swarm review approve [OPTIONS] REVIEW_ID

  Approve a review.

Options:
  --from TEXT     Reviewer session ID  [required]
  --comment TEXT  Approval comment

review reject

Usage: claude-swarm review reject [OPTIONS] REVIEW_ID

  Reject a review.

Options:
  --from TEXT     Reviewer session ID  [required]
  --comment TEXT  Rejection comment

merge-ok

Usage: claude-swarm merge-ok [OPTIONS] SESSION_ID

  Check if a session is clear to merge (all reviews approved).

commit

Usage: claude-swarm commit [OPTIONS] DESCRIPTION

  Commit staged changes with session-prefixed message.

Options:
  --session TEXT  Session ID  [required]

push

Usage: claude-swarm push [OPTIONS]

  Push changes (acquires lock, rebases, pushes, releases).

Options:
  --session TEXT  Session ID  [required]

pull

Usage: claude-swarm pull [OPTIONS]

  Pull and rebase.

Options:
  --session TEXT  Session ID  [required]

git-status

Usage: claude-swarm git-status [OPTIONS]

  Show git status.

git-diff

Usage: claude-swarm git-diff [OPTIONS]

  Show git diff.

Options:
  --staged  Show staged changes

git-log

Usage: claude-swarm git-log [OPTIONS]

  Show recent git log.

orchestrate run

Usage: claude-swarm orchestrate run [OPTIONS]

  Start the orchestrator poll loop.

Options:
  --interval INTEGER       Poll interval in seconds
  --auto-approve           Auto-approve reviews assigned to orchestrator
  --stale-minutes INTEGER  Minutes before a session is considered stale
  --quiet                  Only print when there are events

orchestrate dashboard

Usage: claude-swarm orchestrate dashboard [OPTIONS]

  One-shot status dashboard (sessions + inbox + reviews + claims).
