Documentation

Install

pip install claude-swarm
brew tap laurenalexander2/tap && brew install claude-swarm

Requires Python 3.11+. Dependencies: click, rich.

Getting Started

Quick Start (Recommended)

ao setup                                    # verify Claude Code is installed
ao start "Build a REST API with auth"       # launch orchestrator mode

ao start does the following:

  1. Checks that the Claude Code CLI is installed
  2. Creates the .claude-swarm/ directory
  3. Writes a CLAUDE.md with the full coordination protocol
  4. Launches Claude Code with an orchestrator prompt
  5. The orchestrator decomposes your project into workstreams
  6. Prints copy-pasteable prompts for each session
  7. You paste each prompt into a new Claude Code terminal
  8. The orchestrator monitors all sessions automatically

Manual Setup

# Initialize with your sessions
claude-swarm init --sessions "A:frontend B:backend C:infra"

# Check status
claude-swarm status

Workflow

The recommended process for each session:

  1. Pull before starting
  2. Sync between every task — claude-swarm sync --session ID
  3. Share decisions via context add whenever you make a choice that affects others
  4. Claim files before editing to prevent concurrent edits
  5. Update status when starting, blocking, or completing
  6. Commit often with small, focused commits
  7. Request review when ready to merge
  8. Wait for merge-ok before pushing
  9. Push only via CLI — never raw git push
  10. Never force push

Global Options

claude-swarm --db PATH COMMAND      # specify database path
claude-swarm --version              # print version

You can also set the database path via environment variable:

export CLAUDE_SWARM_DB=".claude-swarm/bus.db"

CLI Reference

sync

The most important command. Run between every task. Checks inbox, reads new shared context, sends heartbeat — all in one call.

claude-swarm sync --session A
--session (required)
Session ID to sync

Output when nothing is new: silent (no output, exit 0). Output when something is new:

[SYNC] 1 new message(s), 2 new context
MSG from=B: "Auth types are exported now"
CTX [decision] by=B: "JWT for auth, not session cookies"
CTX [interface] by=C: "POST /users returns {id, email, created_at}"

context add

Add a shared context entry. Categories: decision, interface, warning, convention, discovery.

claude-swarm context add "API uses snake_case for all fields" --session A --category decision
--session (required)
Session ID adding the context
--category (required)
One of: decision, interface, warning, convention, discovery

context show

View all shared context entries.

claude-swarm context show

message

Send a message to another session.

claude-swarm message B "Can you expose the auth types?" --from A
TO_ID (argument)
Recipient session ID
BODY (argument)
Message text
--from (required)
Sender session ID

inbox

Check unread messages and pending reviews for a session.

claude-swarm inbox --session B
--session (required)
Session ID to check inbox for

reply

Reply to a message. Marks the original as read and sends reply to the original sender.

claude-swarm reply 1 "Done, exported from auth/types.ts" --from B
MESSAGE_ID (argument)
ID of message to reply to
BODY (argument)
Reply text
--from (required)
Sender session ID

init

Initialize the database and register sessions.

claude-swarm init --sessions "A:frontend B:backend C:infra"
--sessions (required)
Space-separated "ID:workstream" pairs

status

Show all sessions and their current status.

claude-swarm status

update

Update a session's status or note.

claude-swarm update A --status blocked --note "waiting on B for types"
SESSION_ID (argument)
Session to update
--status
New status: running, blocked, done
--note
Status note

claim

Claim a file for exclusive editing. Fails if already claimed by another session.

claude-swarm claim src/auth/middleware.py --session A
FILE_PATH (argument)
Path to file to claim
--session (required)
Session ID

claims

List all active file claims.

claude-swarm claims

unclaim

Release a file claim.

claude-swarm unclaim src/auth/middleware.py --session A
FILE_PATH (argument)
Path to file to release
--session (required)
Session ID

review request

Request a review from another session. Sends a notification to the reviewer's inbox.

claude-swarm review request --from A --to B --diff "$(git diff main)"
--from (required)
Requester session ID
--to (required)
Reviewer session ID
--diff (required)
Diff content

review list

List all reviews.

claude-swarm review list

review show

Show details of a specific review including the full diff.

claude-swarm review show 1

review approve

Approve a review. Sends approval notification to the requester.

claude-swarm review approve 1 --from B --comment "lgtm"
REVIEW_ID (argument)
Review ID to approve
--from (required)
Reviewer session ID
--comment (optional, default "")
Approval comment

review reject

Reject a review. Resets status to pending. Sends rejection notification to the requester.

claude-swarm review reject 1 --from B --comment "fix the error types first"
REVIEW_ID (argument)
Review ID to reject
--from (required)
Reviewer session ID
--comment (optional, default "")
Rejection reason

merge-ok

Check if a session is clear to merge. Exit code 0 if all reviews are approved (or no reviews exist). Exit code 1 if any reviews are pending.

claude-swarm merge-ok A

commit

Commit staged changes. Message is auto-prefixed with the session ID.

claude-swarm commit "add PII detection middleware" --session A
# produces: [Session A] add PII detection middleware
DESCRIPTION (argument)
Commit message
--session (required)
Session ID

push

Push changes. Acquires the git lock, rebases on latest, pushes, then releases the lock. On failure, sets session status to "blocked".

claude-swarm push --session A
--session (required)
Session ID

pull

Pull and rebase on latest.

claude-swarm pull --session A
--session (required)
Session ID

git-status

claude-swarm git-status

git-diff

claude-swarm git-diff --staged
--staged (flag)
Show only staged changes

git-log

claude-swarm git-log

orchestrate run

Start the orchestrator poll loop. Detects blocked sessions, stale sessions, pending reviews, and unread messages.

claude-swarm orchestrate run --interval 10 --auto-approve
--interval (default 10)
Poll interval in seconds
--auto-approve (flag)
Auto-approve reviews assigned to orchestrator
--stale-minutes (default 15)
Minutes before a session is considered stale
--quiet (flag)
Only print when there are events

orchestrate dashboard

One-shot status view: all sessions, inbox messages, pending reviews, active file claims.

claude-swarm orchestrate dashboard

ao setup

Check that the Claude Code CLI is installed.

ao setup

ao start

Launch orchestrator mode in Claude Code.

ao start "Build a REST API with auth" --project-dir .
DESCRIPTION (argument)
Project description
--project-dir (default ".")
Project directory
