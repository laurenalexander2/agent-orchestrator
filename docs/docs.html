Documentation

Install

pip install claude-swarm
brew tap laurenalexander2/tap && brew install claude-swarm

Requires Python 3.11+. Dependencies: click, rich.

Getting Started

ao setup                              # check Claude Code is installed
ao start "Build a REST API with auth"  # launch orchestrator mode

ao start creates .claude-swarm/, writes a CLAUDE.md, and launches Claude Code as orchestrator. The orchestrator splits the project into workstreams and prints a prompt for each session. You paste each into a separate terminal.

Manual setup:

claude-swarm init --sessions "A:frontend B:backend C:infra"
claude-swarm status

Workflow

  1. Pull before starting
  2. Sync between every task — claude-swarm sync --session ID
  3. Share decisions via context add when you make choices that affect others
  4. Claim files before editing
  5. Update status when starting, blocking, or completing
  6. Commit often with small, focused commits
  7. Request review when ready to merge
  8. Wait for merge-ok before pushing
  9. Push only via CLI — never raw git push
  10. Never force push

Global Options

claude-swarm --db PATH COMMAND      # specify database path
claude-swarm --version              # print version
export CLAUDE_SWARM_DB=".claude-swarm/bus.db"  # env var alternative

CLI Reference

sync

Check inbox, read new shared context, send heartbeat. Run between every task.

claude-swarm sync --session A

--session (required) — Session ID

Silent when nothing is new. When there is something new:

[SYNC] 1 new message(s), 2 new context
MSG from=B: "Auth types are exported now"
CTX [decision] by=B: "JWT for auth, not session cookies"

context add

Add a shared context entry.

claude-swarm context add "API uses snake_case for all fields" --session A --category decision

--session (required) — Session ID
--category (required) — One of: decision, interface, warning, convention, discovery

context show

claude-swarm context show

message

claude-swarm message B "Can you expose the auth types?" --from A

TO_ID (arg) — Recipient
BODY (arg) — Message text
--from (required) — Sender session ID

inbox

claude-swarm inbox --session B

--session (required) — Session ID

reply

Marks original as read, sends reply to original sender.

claude-swarm reply 1 "Done, exported from auth/types.ts" --from B

MESSAGE_ID (arg) — ID of message to reply to
BODY (arg) — Reply text
--from (required) — Sender session ID

init

claude-swarm init --sessions "A:frontend B:backend C:infra"

--sessions (required) — Space-separated "ID:workstream" pairs

status

claude-swarm status

update

claude-swarm update A --status blocked --note "waiting on B for types"

SESSION_ID (arg) — Session to update
--status — running, blocked, or done
--note — Status note

claim

Claim a file for exclusive editing. Exit 1 if already claimed by another session.

claude-swarm claim src/auth/middleware.py --session A

FILE_PATH (arg) — Path to file
--session (required) — Session ID

claims

claude-swarm claims

unclaim

claude-swarm unclaim src/auth/middleware.py --session A

FILE_PATH (arg) — Path to file
--session (required) — Session ID

review request

Sends a notification to the reviewer's inbox.

claude-swarm review request --from A --to B --diff "$(git diff main)"

--from (required) — Requester session ID
--to (required) — Reviewer session ID
--diff (required) — Diff content

review list

claude-swarm review list

review show

claude-swarm review show 1

review approve

Sends approval notification to requester.

claude-swarm review approve 1 --from B --comment "lgtm"

REVIEW_ID (arg)
--from (required) — Reviewer session ID
--comment (optional) — Approval comment

review reject

Resets status to pending. Sends rejection notification to requester.

claude-swarm review reject 1 --from B --comment "fix the error types first"

REVIEW_ID (arg)
--from (required) — Reviewer session ID
--comment (optional) — Rejection reason

merge-ok

Exit 0 if all reviews approved or no reviews. Exit 1 if any reviews pending.

claude-swarm merge-ok A

commit

Auto-prefixes message with session ID.

claude-swarm commit "add PII detection middleware" --session A
# produces: [Session A] add PII detection middleware

DESCRIPTION (arg) — Commit message
--session (required) — Session ID

push

Acquires git lock, rebases, pushes, releases lock. On failure, sets session status to "blocked".

claude-swarm push --session A

--session (required) — Session ID

pull

claude-swarm pull --session A

--session (required) — Session ID

git-status

claude-swarm git-status

git-diff

claude-swarm git-diff --staged

--staged (flag) — Show only staged changes

git-log

claude-swarm git-log

orchestrate run

Continuous poll loop. Detects blocked/stale sessions, pending reviews, unread messages.

claude-swarm orchestrate run --interval 10 --auto-approve

--interval (default 10) — Poll interval in seconds
--auto-approve (flag) — Auto-approve reviews assigned to orchestrator
--stale-minutes (default 15) — Minutes before session is considered stale
--quiet (flag) — Only print when there are events

orchestrate dashboard

claude-swarm orchestrate dashboard

ao setup

ao setup

ao start

ao start "Build a REST API with auth" --project-dir .

DESCRIPTION (arg) — Project description
--project-dir (default ".") — Project directory
