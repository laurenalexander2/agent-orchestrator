Documentation

Install

pip install claude-swarm
brew tap laurenalexander2/tap && brew install claude-swarm

Requires Python 3.11+.

Getting Started

ao setup                              # check Claude Code is installed
ao start "Build a REST API with auth"  # launch orchestrator mode

Or set up manually:

claude-swarm init --sessions "A:frontend B:backend C:infra"

Workflow

  1. Pull before starting
  2. Sync between every task
  3. Share decisions via context add
  4. Claim files before editing
  5. Update status when blocking or completing
  6. Commit often
  7. Request review when ready
  8. Wait for merge-ok before pushing
  9. Push only via CLI
  10. Never force push

CLI Reference

Usage: claude-swarm [OPTIONS] COMMAND [ARGS]...

Options:
  --version  Show the version and exit.
  --db TEXT  Path to bus database
  --help     Show this message and exit.

Commands:
  claim        Claim a file for exclusive editing.
  claims       List all file claims.
  commit       Commit staged changes with session-prefixed message.
  context      Shared context management commands.
  git-diff     Show git diff.
  git-log      Show recent git log.
  git-status   Show git status.
  inbox        Check unread messages for a session.
  init         Initialize the orchestrator database and register sessions.
  merge-ok     Check if a session is clear to merge (all reviews approved).
  message      Send a message to another session.
  orchestrate  Orchestrator mode — monitor and coordinate sessions.
  pull         Pull and rebase.
  push         Push changes (acquires lock, rebases, pushes, releases).
  reply        Reply to a message (marks original as read and sends reply).
  review       Review management commands.
  status       Show all sessions and their status.
  sync         Sync: check inbox, read new context, heartbeat.
  unclaim      Release a file claim.
  update       Update a session's status or note.

The --db option can also be set via CLAUDE_SWARM_DB environment variable.

sync

Usage: claude-swarm sync [OPTIONS]

  Sync: check inbox, read new context, heartbeat. Silent when nothing new.

Options:
  --session TEXT  Session ID to sync  [required]
$ claude-swarm sync --session A
[SYNC] 1 new message(s), 2 new context
MSG from=B: "Auth types are exported now"
CTX [decision] by=B: "JWT for auth, not session cookies"

context

Usage: claude-swarm context [OPTIONS] COMMAND [ARGS]...

  Shared context management commands.

Commands:
  add   Add a shared context entry.
  show  Show all shared context entries.

context add

Usage: claude-swarm context add [OPTIONS] BODY

  Add a shared context entry.

Options:
  --session TEXT                  Session ID adding the context  [required]
  --category [decision|interface|warning|convention|discovery]
                                  Context category  [required]
$ claude-swarm context add "API uses snake_case for all fields" --session A --category decision

context show

Usage: claude-swarm context show [OPTIONS]

  Show all shared context entries.

message

Usage: claude-swarm message [OPTIONS] TO_ID BODY

  Send a message to another session.

Options:
  --from TEXT  Sender session ID  [required]
$ claude-swarm message B "Can you expose the auth types?" --from A

inbox

Usage: claude-swarm inbox [OPTIONS]

  Check unread messages for a session.

Options:
  --session TEXT  Session ID to check inbox for  [required]

reply

Usage: claude-swarm reply [OPTIONS] MESSAGE_ID BODY

  Reply to a message (marks original as read and sends reply).

Options:
  --from TEXT  Sender session ID  [required]
$ claude-swarm reply 1 "Done, exported from auth/types.ts" --from B

  Initialize the orchestrator database and register sessions.

Options:
  --sessions TEXT  Space-separated "ID:workstream" pairs  [required]
$ claude-swarm init --sessions "A:frontend B:backend C:infra"

review

Usage: claude-swarm review [OPTIONS] COMMAND [ARGS]...

  Review management commands.

Commands:

  list     List all reviews.
  reject   Reject a review.

  show     Show details of a specific review.

review request

Usage: claude-swarm review request [OPTIONS]

  Request a review from another session.

Options:
  --from TEXT  Requester session ID  [required]
  --to TEXT    Reviewer session ID  [required]
  --diff TEXT  Diff content  [required]
$ claude-swarm review request --from A --to B --diff "$(git diff main)"

review list

Usage: claude-swarm review approve [OPTIONS] REVIEW_ID

  Approve a review.

Options:

review reject

Usage: claude-swarm review reject [OPTIONS] REVIEW_ID

merge-ok

Usage: claude-swarm merge-ok [OPTIONS] SESSION_ID

  Check if a session is clear to merge (all reviews approved).

Exit 0 if clear. Exit 1 if blocked.

commit

Usage: claude-swarm commit [OPTIONS] DESCRIPTION

  Commit staged changes with session-prefixed message.

Options:
  --session TEXT  Session ID  [required]

Committed: [Session A] add PII detection middleware

push

Usage: claude-swarm push [OPTIONS]

  Push changes (acquires lock, rebases, pushes, releases).

Options:

orchestrate

Usage: claude-swarm orchestrate [OPTIONS] COMMAND [ARGS]...

  Orchestrator mode — monitor and coordinate sessions.

Commands:
  dashboard  One-shot status dashboard (sessions + inbox + reviews + claims).
  run        Start the orchestrator poll loop.

orchestrate run

Usage: claude-swarm orchestrate run [OPTIONS]

  Start the orchestrator poll loop.

Options:
  --interval INTEGER       Poll interval in seconds
  --auto-approve           Auto-approve reviews assigned to orchestrator
  --stale-minutes INTEGER  Minutes before a session is considered stale
  --quiet                  Only print when there are events
$ claude-swarm orchestrate run --interval 10 --auto-approve

orchestrate dashboard

Usage: claude-swarm orchestrate dashboard [OPTIONS]

  One-shot status dashboard (sessions + inbox + reviews + claims).
