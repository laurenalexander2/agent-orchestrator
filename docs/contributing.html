Contributing

Setup

git clone https://github.com/laurenalexander2/agent-orchestrator.git
cd claude-swarm
pip install -e .
pytest tests/ -v

Requires Python 3.11+. No external databases or services needed.

Workflow

  1. Fork the repo and create a feature branch
  2. Write tests first (TDD) — add to the appropriate tests/test_*.py file
  3. Implement the feature
  4. Run pytest tests/ -v — all tests must pass
  5. Open a PR

Where to Put Things

ModuleResponsibility
bus.pyAll SQLite reads/writes
git.pyGit operations + push lock
merge.pyConflict self-healing + file owner notifications
orchestrator.pyPoll loop, blocked/stale detection, auto-approve
cli.pyClick CLI commands (thin layer)
ao.pyQuick-start entry point

New features probably belong in bus.py (data) or a new module (logic), with a CLI surface in cli.py.

Guidelines

  • Keep it simple. This is a coordination tool, not a framework.
  • Every feature needs tests. No exceptions.
  • CLI commands should be self-documenting with clear --help text.
  • SQLite is the only dependency for state. No external databases, no Redis, no message queues.

Tests

pytest tests/ -v

90 tests covering shared context, sync, messaging, reviews, file claims, git lock, conflict self-healing, orchestrator, CLI integration, and template generation.

Roadmap

High-impact features for v2:

  • Claude Code hooks integration — automatic sync via PreToolUse hooks
  • Task queue — shared backlog that sessions pull from
  • Session dependencies — DAG of task dependencies for sequencing
  • Deadlock detection — identify circular blocking between sessions
  • Per-session branches — each session on its own branch with PR workflow
  • Dynamic scaling — add/remove sessions while the swarm is running
