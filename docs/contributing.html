Contributing

Development Setup

git clone https://github.com/laurenalexander2/claude-swarm.git
cd claude-swarm
pip install -e .
pytest tests/ -v

Requires Python 3.11+. No external databases or services needed — everything runs on SQLite.

Development Workflow

  1. Fork the repo and create a feature branch
  2. Write tests first (TDD) — add them to the appropriate tests/test_*.py file
  3. Implement the feature
  4. Run pytest tests/ -v and make sure all tests pass
  5. Open a PR

Code Structure

If you're adding a new feature, it probably belongs in bus.py (data) or a new module (logic), with a CLI surface in cli.py.

ModuleResponsibility
bus.pyAll SQLite reads/writes — sessions, messages, reviews, file claims, shared context, sync
git.pyGit operations + push lock management
merge.pyConflict self-healing + file owner notifications
orchestrator.pyPoll loop, blocked/stale detection, auto-approve
cli.pyClick CLI commands (thin layer over the other modules)
ao.pyQuick-start entry point (ao setup, ao start)

Guidelines

  • Keep it simple. This is a coordination tool, not a framework.
  • Every feature needs tests. No exceptions. The project currently has 90 tests.
  • CLI commands should be self-documenting. Use --help text and clear naming.
  • SQLite is the only dependency for state. No external databases, no Redis, no message queues.

Testing

pytest tests/ -v

90 tests covering:

  • Shared context CRUD and category validation
  • Sync (inbox + context + heartbeat in one call)
  • Message exchange and inbox filtering
  • Review gating (blocks merge until approved, rejection resets to pending)
  • Review notifications (reviewer gets request, requester gets approval/rejection)
  • File claim conflicts and release semantics
  • Git lock acquire/release and concurrent push safety
  • Commit message prefixing with session ID
  • Conflict self-healing for non-overlapping changes
  • Orchestrator poll loop (blocked/stale detection, auto-approve)
  • CLI integration flows
  • Template generation (CLAUDE.md, session prompts, orchestrator prompt)

Roadmap

High-impact contributions for v2:

  • Claude Code hooks integration — automatic sync via PreToolUse hooks, making coordination fully transparent to the agent
  • Task queue — shared backlog that sessions pull from instead of fixed assignments
  • Session dependencies — DAG of task dependencies so the orchestrator can sequence work
  • Deadlock detection — identify circular blocking between sessions
  • Per-session branches — each session works on its own branch with PR workflow
  • Dynamic scaling — add/remove sessions while the swarm is running

Reporting Issues

Open an issue at github.com/laurenalexander2/claude-swarm/issues with:

  • What you expected to happen
  • What actually happened
  • Steps to reproduce
  • Output of claude-swarm status if relevant
