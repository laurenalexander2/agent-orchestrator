claude-swarm

Coordinate parallel Claude Code sessions. Shared context, messaging, file claims, peer reviews, and a serialized git lock — all backed by a single SQLite database.

pip install claude-swarm

brew tap laurenalexander2/tap && brew install claude-swarm

The Problem

When you run multiple Claude Code sessions on the same codebase, things break fast:

  • Sessions overwrite each other's files — two agents editing the same file at the same time leads to lost work
  • Decisions don't propagate — one session picks JWT for auth, another picks session cookies, and they only find out when things break
  • No communication channel — sessions can't ask each other questions, report blockers, or request help
  • No quality gate — code gets pushed without anyone reviewing it, including the other sessions that depend on it
  • Git pushes collide — concurrent rebases and pushes produce conflicts that waste time to untangle

How claude-swarm Solves It

Shared Context

A team wiki all sessions read and write. Share decisions, interfaces, warnings, and conventions. Everyone stays aligned.

Messaging

Session-to-session messages with inbox tracking. Ask questions, share status updates, report blockers. The sync command checks everything in one call.

File Claims

Claim files before editing. Exclusive ownership prevents two sessions from touching the same file. Release when done.

Peer Review

Request reviews, approve or reject, and gate merges behind peer approval. No code ships without someone checking it.

Git Lock

Serialize pushes through a single lock. Acquire, rebase, push, release. No more concurrent push collisions.

Orchestrator Mode

A central monitor that watches all sessions, detects blocked and stale agents, handles reviews, and keeps the swarm moving.

Quick Start

# Check that Claude Code is installed
ao setup

# Launch orchestrator mode with your project description
ao start "Build a REST API with auth and rate limiting"

This writes a CLAUDE.md with the full coordination protocol, creates the .claude-swarm/ directory, and launches Claude Code as the orchestrator. The orchestrator decomposes your project into workstreams and prints copy-pasteable prompts — one for each session. Paste each into a separate Claude Code terminal and they coordinate automatically.

How It Works

ao start "Build X"
     |
     v
Claude Code (orchestrator terminal)
     |
     +-- Plans workstreams
     +-- Runs: claude-swarm init --sessions "..."
     +-- Prints copy-paste prompts for each session
     +-- Starts: claude-swarm orchestrate run
           |
           v
     Monitors all sessions via SQLite bus
           |
     +-----+-----+
     v           v
Session A    Session B      (each in its own terminal)
     |           |
     +-- sync    +-- sync     (inbox + context + heartbeat)
     +-- claim   +-- claim    (exclusive file access)
     +-- context +-- context  (share decisions)
     +-- commit  +-- commit   (session-prefixed)
     +-- review  +-- review   (peer approval)
     +-- push    +-- push     (lock-serialized)
