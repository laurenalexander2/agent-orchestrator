claude-swarm

A CLI for coordinating multiple Claude Code sessions working on the same codebase. All state is stored in a single SQLite database.

pip install claude-swarm
brew tap laurenalexander2/tap && brew install claude-swarm

Requires Python 3.11+.

What It Does

When multiple Claude Code sessions work on the same repo, they need a way to coordinate. claude-swarm provides:

  • Shared context — a team wiki where sessions share decisions, interfaces, warnings, and conventions
  • Sync — one command that checks inbox, reads new context, and sends a heartbeat
  • Messaging — session-to-session messages with read/unread tracking
  • File claims — exclusive file ownership so two sessions don't edit the same file
  • Peer review — review requests, approval/rejection, merge gating
  • Git lock — serialized pushes so concurrent rebases don't collide
  • Orchestrator — a monitor that detects blocked/stale sessions and keeps things moving

Quick Start

ao setup                              # check Claude Code is installed
ao start "Build a REST API with auth"  # launch orchestrator mode

This creates a .claude-swarm/ directory, writes a CLAUDE.md with the coordination protocol, and launches Claude Code as orchestrator. The orchestrator splits the project into workstreams and prints a prompt for each session. You paste each prompt into a separate Claude Code terminal.

Or set up manually:

claude-swarm init --sessions "A:frontend B:backend C:infra"
claude-swarm status

How It Works

ao start "Build X"
     |
     v
Claude Code (orchestrator terminal)
     |
     +-- Plans workstreams
     +-- Runs: claude-swarm init --sessions "..."
     +-- Prints copy-paste prompts for each session
     +-- Starts: claude-swarm orchestrate run
           |
           v
     Monitors all sessions via SQLite bus
           |
     +-----+-----+
     v           v
Session A    Session B      (each in its own terminal)
     |           |
     +-- sync    +-- sync     (inbox + context + heartbeat)
     +-- claim   +-- claim    (exclusive file access)
     +-- context +-- context  (share decisions)
     +-- commit  +-- commit   (session-prefixed)
     +-- review  +-- review   (peer approval)
     +-- push    +-- push     (lock-serialized)
