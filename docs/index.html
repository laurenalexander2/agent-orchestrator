Because no agent
should have to code alone.

Coordinate multiple Claude Code sessions on the same codebase. Shared context, messaging, file claims, peer review, and a serialized git lock.

$ pip install claude-swarm Click to copy

or: brew tap laurenalexander2/tap && brew install claude-swarm

Quick Start

$ ao start "Build a REST API with auth"

That's it. One command. The orchestrator scans your codebase, proposes a plan, and waits for your approval. Then it prints a prompt for each agent session — paste each into a separate Claude Code terminal and they coordinate automatically.

1
Orchestrator scans your codebase

Reads your project structure, dependencies, and existing code to understand what's there.

2
Proposes a plan & splits into workstreams

Decomposes the task into parallel sessions — e.g. frontend, backend, tests — and waits for your approval.

3
Prints a prompt per session

Each prompt contains the agent's role, claimed files, and the full coordination protocol. Paste into a Claude Code terminal.

4
Agents sync, message, and make decisions together

All state lives in a single SQLite database. Agents sync between tasks via hooks, message each other through a shared bus, and broadcast decisions as shared context.

5
Peer review and a linear history

Agents claim files before editing, review each other's changes, and commit through a serialized lock. No race conditions, no merge conflicts, no rebases, and no commits unchecked.

Coordination

All state lives in a single SQLite database. No Redis, no message queues, no external services. Agents stay in sync by polling between tasks.

Sync

ao sync --session A

The most important command. Run between every task. Does three things in one call: checks inbox for new messages, reads shared context entries since last sync, and sends a heartbeat so the orchestrator knows the session is alive. Silent when nothing's new.

Message

ao message B "Can you expose the auth types?" --from A

Send a direct message to another agent through the shared bus. Reply with ao reply 1 "Done, exported from auth/types.ts" --from B. Check unread messages anytime with ao inbox --session A.

Share Context

ao context add "JWT for auth, not session cookies" --session B --category decision

Broadcast decisions, interfaces, warnings, conventions, and discoveries to all agents. Categories keep context organized. Every agent sees new entries on their next sync. View everything with ao context show.

Status Updates

ao update A --status blocked --note "waiting on B for types"

Set your session status to running, blocked, or done. Add notes so the orchestrator and other agents know what's happening. View all sessions with ao status.

Code & Ship

File ownership, peer review, and serialized commits. Agents follow the same workflow as human teams — just faster.

Claim Files

ao claim src/auth.ts --session A

Claim a file for exclusive editing. Exits with an error if another agent already holds the claim. View all claims with ao claims, release with ao unclaim src/auth.ts --session A.

Peer Review

ao review request --session A --reviewer B

Request a code review from another agent before merging. The reviewer can approve with ao review approve, reject with ao review reject, or leave comments. View pending reviews with ao review list.

Commit & Push

ao commit --session A -m "Add auth middleware"

Commit through the serialized git lock — one agent at a time. Clean, linear history with no race conditions. Wait for ao merge-ok --session A before pushing. Push with ao push --session A — never raw git push.
