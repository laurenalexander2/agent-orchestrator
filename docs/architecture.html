Architecture

System Overview

ao start "Build X"
     |
     v
Claude Code (orchestrator terminal)
     |
     +-- Plans workstreams
     +-- Runs: claude-swarm init --sessions "..."
     +-- Prints copy-paste prompts for each session
     +-- Starts: claude-swarm orchestrate run
           |
           v
     Monitors all sessions via SQLite bus
           |
     +-----+-----+
     v           v
Session A    Session B      (each in its own terminal)
     |           |
     +-- sync    +-- sync     (inbox + context + heartbeat)
     +-- claim   +-- claim    (exclusive file access)
     +-- context +-- context  (share decisions)
     +-- commit  +-- commit   (session-prefixed)
     +-- review  +-- review   (peer approval)
     +-- push    +-- push     (lock-serialized)

Every session is a separate Claude Code terminal. The only shared state is the SQLite database at .claude-swarm/bus.db. Sessions coordinate by reading and writing to this database through the CLI.

Modules

claude_swarm/
+-- bus.py            # SQLite message bus
+-- git.py            # Git operations + push lock
+-- merge.py          # Conflict self-healing
+-- orchestrator.py   # Poll loop + monitoring
+-- cli.py            # Click CLI commands
+-- ao.py             # Quick-start entry point
ModuleResponsibility
bus.pyAll SQLite reads/writes — sessions, messages, reviews, file claims, shared context, sync
git.pyGit operations (add, commit, push, pull, status, diff, log) + push lock management
merge.pyConflict self-healing for non-overlapping changes. Notifies file owners when their claimed files are touched.
orchestrator.pyPoll loop that monitors all sessions. Detects blocked/stale sessions, pending reviews, unread messages. Auto-approve for orchestrator-assigned reviews.
cli.pyThin Click CLI layer. All commands delegate to bus.py, git.py, or orchestrator.py.
ao.pyQuick-start entry point. ao setup checks prerequisites, ao start generates templates and launches Claude Code.

Design Decisions

  • SQLite is the only state store. No Redis, no message queues, no external services. One file with WAL mode handles all concurrency.
  • Copy-paste prompts, not sub-agents. The orchestrator prints prompts you paste into separate terminals. Each session is a full Claude Code instance with full autonomy, not a constrained sub-agent.
  • Sync is the core primitive. One command does inbox + context + heartbeat. Sessions run it between every task.
  • File claims are advisory. The CLI enforces claims, but nothing stops a session from editing a file it hasn't claimed. Claims are a coordination signal, not a hard lock.
  • Reviews gate merges, not commits. Sessions commit freely. The review gate only blocks merge-ok, which is checked before pushing.

Database Schema

6 tables in .claude-swarm/bus.db. SQLite with WAL journal mode and foreign keys enabled.

sessions

CREATE TABLE sessions (
    id          TEXT PRIMARY KEY,
    workstream  TEXT NOT NULL,
    status      TEXT DEFAULT 'running',
    branch      TEXT,
    note        TEXT,
    updated_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    last_sync_at DATETIME
);

id — e.g. "A", "B", "orchestrator"
status — "running", "blocked", "done"
updated_at — refreshed on every sync and status update
last_sync_at — context entries after this timestamp are returned on next sync. NULL means first sync returns all context.

messages

CREATE TABLE messages (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    from_id     TEXT NOT NULL,
    to_id       TEXT NOT NULL,
    body        TEXT NOT NULL,
    type        TEXT DEFAULT 'message',
    status      TEXT DEFAULT 'unread',
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);

type — "message", "review_request", "review_approved", "review_rejected"
status — "unread" or "read". Sync marks all unread as read.

reviews

CREATE TABLE reviews (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    requester   TEXT NOT NULL,
    reviewer    TEXT NOT NULL,
    diff        TEXT,
    status      TEXT DEFAULT 'pending',
    comments    TEXT,
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);

status — "pending" or "approved". Rejection resets to "pending".
merge-ok returns true only when zero reviews have status != "approved".

git_lock

CREATE TABLE git_lock (
    id          INTEGER PRIMARY KEY CHECK (id = 1),
    held_by     TEXT,
    acquired_at DATETIME
);

Single-row table. held_by is session ID or NULL. Push acquires, rebases, pushes, releases. Others wait.

file_claims

CREATE TABLE file_claims (
    file_path   TEXT PRIMARY KEY,
    session_id  TEXT NOT NULL,
    claimed_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);

One owner per file. Claiming fails if a different session holds it. Same session can re-claim.

shared_context

CREATE TABLE shared_context (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id  TEXT NOT NULL,
    category    TEXT NOT NULL,
    body        TEXT NOT NULL,
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);

category — "decision", "interface", "warning", "convention", "discovery"
Timestamps use Python microsecond precision for sync filtering.
